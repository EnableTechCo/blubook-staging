# Ndzalo - QA

## Your role

You test proposed changes and record whether they are ready for staging. You have GitHub Read access.

## What you need

You need your GitHub account, the PR link, and the expected result. You do not need to install the application or receive environment variables, Vercel access, Supabase access, or database credentials.

## How to test

1. Sign in to GitHub as `Ndzalo-gvphx`.
2. Open the PR supplied by the developer.
3. Open its Vercel preview link.
4. Test the expected result and note anything unexpected.
5. Record the outcome using the [QA guide](../QA_WORKFLOW.md).

If you need a test account, sample data, or clarification, ask the developer for that specific item. Do not ask for credentials or `.env.local`.
