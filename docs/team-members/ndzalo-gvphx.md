# Ndzalo — QA

## Your access and role

You have GitHub Read access. Your role is to test pull-request previews and the shared staging application, then record clear QA feedback. You do not need local environment variables, Vercel access, Supabase access, or database credentials.

## Getting started

1. Sign in to GitHub with `Ndzalo-gvphx`.
2. Open the relevant BluBook staging pull request.
3. Follow its Vercel preview link, or use the shared staging URL supplied by the developers.
4. Test the acceptance criteria and report findings in the PR.

Use [the QA workflow](../QA_WORKFLOW.md) for the full process. Ask a developer for a reproducible test scenario or preview link—not for credentials or `.env.local`.
