# Ntokozo - UX design

## Your role

You review the user experience and interface quality of proposed changes. You have GitHub Read access.

## What you need

You need your GitHub account and the PR or staging link supplied by the developer. You do not need to install the application or receive environment variables, Vercel access, Supabase access, or database credentials.

## How to review

1. Sign in to GitHub as `ntokozo632`.
2. Open the PR supplied by the developer.
3. Open its Vercel preview link.
4. Review the screen against the agreed design and intended user journey.
5. Add focused feedback in the PR. Screenshots and annotations are helpful.

Ask the developer for a preview link or the expected design outcome. Never request or store environment values.
